# Writing Github Issues Skill

## Instructions
Follow these core patterns and best practices when writing new issues, bugs tasks, features in Github:
- When creating issues in Gihub, ensure to provide clear and concise titles that accurately reflect the task or feature.
- Keep it lean, simple, and DRY (Don't Repeat Yourself). Avoid unnecessary complexity and duplication in the issue description.
- Unless instructed, avoid too many implementation details; focus on the "what" and "why" rather than the "how".
- Use Markdown formatting to enhance readability. Utilize headings, bullet points, and code blocks where appropriate.
- Before creating an issue, check for existing similar issues to avoid duplication.
- Structure the issue with clear sections such as "Scope of Work", "Out of Scope", "Acceptance Criteria", "Reference Implementation", and "Technical Notes" to provide comprehensive context.
- Before actually creating an issue, request feedback.
